# Reflix — 우측 패널 Inspector 재디자인

## 개요

Reflix의 우측 패널을 Eagle 스타일의 inspector 분위기로 재구성한다. 이번 작업의 목표는 기능 추가가 아니라, 현재 `selectedClipId` 기반의 읽기 전용 상세 패널을 더 조밀하고 명확한 정보 계층으로 재배치하는 것이다.

기준 레퍼런스는 사용자가 공유한 Eagle 우측 패널 스크린샷이며, 최종 방향은 그중 `A. Eagle Lite` 안으로 확정되었다.

## 목표

- 현재 우측 패널을 Eagle 같은 "dark inspector" 분위기로 재정렬한다.
- 기존 데이터 소스(`public/data/clips/*.json`)는 그대로 사용한다.
- 편집 기능 없이 읽기 전용으로 유지한다.
- 정보 순서를 `미리보기 → 팔레트 → 제목/메모/링크 → 폴더 → 태그 → 속성 → 공유`로 고정한다.

## 비목표

- 태그, 폴더, 메모, URL 편집 기능 추가
- 공유 기능의 실제 동작 구현
- 추가 메타데이터 확장
- 우측 패널 외 레이아웃 변경

## 핵심 결정사항

| 항목 | 결정 |
|---|---|
| 디자인 방향 | Eagle Lite |
| 패널 성격 | 읽기 전용 inspector |
| 시각 톤 | 진한 회색 계열, 둥근 카드, 얇은 섹션 구분선 |
| 상단 미리보기 | 큰 썸네일/프리뷰 영역 유지 |
| 팔레트 표시 | 미리보기 아래에 pill 형태 스와치 줄 |
| 폴더 위치 | 태그보다 위 |
| 속성 항목 | `평가`, `지속 시간`, `파일 형식`만 표시 |
| 파일 형식 표기 | 확장자 대신 `동영상` 또는 `이미지` |
| 하단 CTA | `내보내기` 대신 `공유` |
| 기능 범위 | 스타일/레이아웃만, 상호작용 추가 없음 |

## 레이아웃 구조

우측 패널은 하나의 세로 inspector 카드처럼 보이게 구성한다.

1. 상단 미리보기
2. 색상 팔레트
3. 제목
4. 메모
5. 링크
6. 폴더 섹션
7. 태그 섹션
8. 속성 섹션
9. 하단 공유 버튼

각 섹션 사이는 얇은 divider로 분리하고, 입력창처럼 보이는 요소도 실제로는 read-only 표시 컴포넌트로 렌더링한다.

## 정보 매핑

### 제목

- 우선순위: `clip.i18n.title[lang] || clip.name`
- 한 줄 또는 최대 두 줄까지 허용할 수 있으나, 기본 디자인은 한 줄 중심으로 맞춘다.

### 메모

- 소스: `clip.annotation`
- 값이 있으면 본문 텍스트 표시
- 값이 없으면 비어 있는 입력창 같은 muted placeholder 스타일 표시

### 링크

- 소스: `clip.url`
- 값이 있으면 URL 문자열 표시
- 값이 없으면 `링크 없음` 또는 이에 준하는 muted 문구 표시

### 폴더

- 소스: `clip.folders`
- 태그 칩과 비슷한 pill 형태로 표시
- 태그보다 먼저 나온다
- 현재 데이터는 폴더 ID 배열이므로, 구현 시 사람이 읽을 수 있는 이름 매핑이 가능하면 이름을 쓰고, 아직 매핑이 없으면 우선 안전한 fallback이 필요하다
- planning 단계에서 폴더 ID를 사람이 읽을 수 있는 레이블로 바꾸는 경로를 결정해야 한다

### 태그

- 소스: `clip.tags`
- 기존처럼 pill/chip 형태 유지
- Eagle 레퍼런스처럼 촘촘한 밀도로 배치
- 클릭 가능해 보이더라도 이번 범위에서는 읽기 전용이다

### 색상 팔레트

- 소스: `clip.palettes`
- 현재처럼 원형 스와치 사용
- 미리보기 바로 아래에 pill 컨테이너 안에 배치
- 5~6개 정도만 노출하고 나머지는 숨겨도 된다

### 속성

속성은 아래 3개만 노출한다.

1. `평가`
   - 소스: `clip.star`
   - 별 문자 또는 기존 별 아이콘 표현 유지
2. `지속 시간`
   - 소스: `clip.duration`
   - `00:08` 같은 시:분:초 포맷으로 정규화
3. `파일 형식`
   - 소스: `clip.ext`
   - 출력값은 확장자가 아니라 사용자 언어 기준의 미디어 타입
   - 예: `mp4`, `mov`, `webm` 계열이면 `동영상`
   - 예: `png`, `jpg`, `jpeg`, `webp`, `gif` 계열이면 `이미지`

## 시각 디자인 기준

### 패널

- 배경은 기존 단색보다 더 깊은 회색 계열 gradient 사용
- 바깥 컨테이너는 둥근 모서리와 은은한 그림자를 가진다
- 내부 아이템은 밝은 텍스트, 낮은 대비의 구분선, 약한 테두리를 사용한다

### 미리보기

- 패널에서 가장 큰 시각적 무게를 가진다
- 직사각형 프리뷰 카드 + 작은 포맷 배지 조합을 유지한다
- 현재 썸네일/영상 로딩 체계를 바꾸지 않는다

### 필드형 블록

- 제목/메모/링크는 Eagle의 입력 필드처럼 보이되, 실제 `<input>` 편집 UX는 아니다
- 각 블록은 동일한 높이, 동일한 radius, 어두운 배경 톤을 공유한다

### 섹션 라벨

- `폴더`, `태그`, `속성`은 muted 소제목으로 표시한다
- 본문보다 작고 대비를 낮춘다

### CTA

- 패널 하단의 단일 버튼 라벨은 `공유`
- 동작은 추후 정의 가능하지만, 이번 범위에서는 시각적 버튼만 제공한다

## 동작 및 상태

### 클립 미선택 상태

- 현재처럼 패널 전체를 숨길지, 빈 상태를 보여줄지는 구현 시점에 기존 UX를 따른다
- 이번 디자인 변경은 선택된 클립이 있을 때의 레이아웃에만 집중한다

### 언어

- 제목은 현재 로케일 우선
- `파일 형식` 레이블 값도 현 로케일에 맞게 노출할 수 있도록 구현 여지를 남긴다
- 다만 이번 spec의 승인된 한국어 표현은 `동영상`, `이미지`다

### 상호작용

- 태그/폴더/공유 버튼의 실제 기능 확장은 이번 범위 밖
- hover 스타일 정도는 허용되지만, 편집/추가/삭제 상태는 만들지 않는다

## 구현 경계

변경 대상은 우측 패널 관련 컴포넌트와, 필요 시 그 표시를 위한 소규모 포맷 헬퍼에 한정한다.

예상 영향 범위:

- `src/components/layout/RightPanel.tsx`
- `src/components/layout/RightPanelContent.tsx`
- 필요 시 `src/app/globals.css`
- 필요 시 미디어 타입/시간 포맷용 소규모 유틸

변경하지 않는 범위:

- 그리드 카드 선택 로직
- 클립 JSON 스키마
- 좌측 패널 / 상단 툴바
- 서버/데이터 파이프라인

## 계획 단계에서 풀어야 할 구현 질문

1. 폴더 ID를 사용자용 레이블로 어떻게 매핑할지
2. `clip.duration`을 inspector용 포맷으로 어디서 변환할지
3. `clip.ext`를 `동영상/이미지`로 바꾸는 헬퍼 위치를 어디에 둘지
4. 패널 폭을 현재 `w-72`에서 유지할지, Eagle 분위기를 위해 약간 넓힐지

## 테스트 관점

- 클립 선택 시 새 레이아웃이 안정적으로 렌더링되는지 확인
- `annotation`, `url`, `palettes`가 비어 있을 때 레이아웃이 깨지지 않는지 확인
- `ext`별 파일 형식 라벨이 기대값으로 나오는지 확인
- `duration` 포맷이 1자리 초, 분 단위, 긴 영상에서도 일관되는지 확인
- 다크 테마 기본 화면에서 대비가 지나치게 약하지 않은지 확인

## 승인 결과

사용자는 아래 조건으로 디자인을 승인했다.

- 방향은 `A. Eagle Lite`
- `속성`은 `평가`, `지속 시간`, `파일 형식`만 사용
- `파일 형식`은 `mp4` 같은 확장자가 아니라 `동영상` 또는 `이미지`
- `폴더`는 `태그` 위에 배치
- 하단 버튼 라벨은 `공유`
