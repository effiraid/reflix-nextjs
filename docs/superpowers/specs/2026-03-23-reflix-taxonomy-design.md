# Reflix — 폴더/태그 분류 체계 설계

## 개요

Reflix의 콘텐츠 분류 체계를 정의한다. Eagle 라이브러리의 폴더/태그 구조를 웹 플랫폼에 1:1 매핑하여, 관리자(Eagle)와 사용자(웹) 양쪽에서 동일한 구조를 사용한다.

**원칙:**
- 폴더 = "캐릭터가 뭘 하고 있나" (동작 유형, 1차 탐색 축)
- 태그 = "어떤 특성을 가졌나" (교차 속성, 2차 필터 축)
- 검색 = 롱테일 (저빈도 태그 + 이름 기반)

## 3탭 구조

| 탭 | 데이터 소스 | 아이템 수 | 설명 |
|---|---|---|---|
| **연출** | 게임,연출.library | 공유 | 시네마틱/영화/애니메이션 연출 레퍼런스 |
| **게임** | 게임,연출.library | 공유 | 게임 엔진 애니메이션 클립 |
| **몬스터** | 동물(몬스터,크리처).library | 1,577 | 크리처/동물/몬스터 전용 |

- **연출/게임**: 같은 데이터풀 (7,258개). 탭 전환 = 좌측 패널의 폴더/태그 그룹 프리셋 변경. 클립은 양쪽 탭에 동시 등장 가능.
- **몬스터**: 완전 별도 Eagle 라이브러리. 독립 데이터셋.
- 탭 판별: 클립 이름 접두어 "연출" / "게임" + 폴더 소속으로 자동 분류. 별도 필드 불필요.

## 폴더 구조 (Eagle = 웹 1:1)

### 연출+게임 라이브러리

4개 색상 섹션, 10개 최상위 카테고리, 최대 2뎁스.

```
── 전투 ──────────────────── 🔴 red
│
├── 이동
│   ├── 걷기
│   ├── 달리기
│   ├── 전력질주
│   ├── 점프
│   └── 기타 (회피, 구르기, 슬라이딩 등)
│
├── 교전
│   ├── 근접 공격
│   ├── 원거리 공격
│   ├── 필살기
│   ├── 충전 (차지)
│   ├── 전투 준비
│   └── 버프/유틸
│
├── 피격
│   ├── 피격
│   ├── 기절
│   └── 사망
│
├── 리턴
│
└── 대기
    ├── 일반
    └── 전투

── 연기 ──────────────────── 🔵 blue
│
├── 대화
│   ├── 대화 (연출)
│   ├── 대화 (인게임)
│   ├── 표정
│   ├── 감정표현 (인게임)
│   └── 무빙홀드
│
├── 리액션
│   ├── 놀람/충격
│   ├── 고통/쓰러짐
│   └── 감정 표출
│
└── 생활 동작
    ├── 앉기/눕기/일어나기
    ├── 잡기/놓기/들기
    └── 기타 (인터랙션 포함)

── 연출 ──────────────────── 🟢 green
│
├── 연출 기법
│   ├── 효과
│   └── 전환
│
└── 게임 연출
    ├── 등장
    ├── 선택
    └── 승리

── 장비 ──────────────────── 🟡 yellow
│
├── 무기별
│   ├── 검
│   ├── 단검
│   ├── 둔기
│   ├── 창/봉
│   ├── 방패
│   ├── 마법
│   ├── 활
│   ├── 총
│   └── 신체 (격투가)
│
└── 포즈
    └── 프리셋
```

**섹션별 역할:**
- **전투 🔴**: 게임 스테이트 머신의 핵심 상태 (이동, 교전, 피격, 대기)
- **연기 🔵**: 비전투 캐릭터 연기 (대화, 리액션, 생활 동작)
- **연출 🟢**: 카메라/편집/프레젠테이션 기법
- **장비 🟡**: 무기별 교차 분류 + 정적 포즈

**기존 Eagle 대비 변경점:**

| 변경 | 이유 |
|---|---|
| "연출(영상)" 폴더 삭제 | 3,858개의 메가폴더. 탭으로 대체, 클립은 동작 폴더로 재분류 |
| "클립 분류" 최상위 폴더 삭제 | 탭 자체가 분류 역할. 불필요한 1뎁스 제거 |
| "대화 그룹" 해체 | 대기/감정표현이 대화와 무관. 대화·대기·감정표현 각각 독립 |
| 교전 > 공격을 근접/원거리로 세분화 | 1,795개 과대. 사용자 시뮬레이션에서 friction 발생 |
| 리턴을 교전 밖으로 분리 | 1,103개로 독립 카테고리 규모 |
| "뽑기" 삭제 | 0개. "승리"에 통합 |
| "리액션" 신설 | orphan 감정 리액션 클립 흡수 (~880개) |
| "생활 동작" 신설 | orphan 일상 동작/인터랙션 클립 흡수 (~550개) |
| 대기를 한 곳에 통합 | 기존 3곳 중복 (대화그룹, 포즈, 교전) → 1곳 |

### 몬스터 라이브러리

```
── 폴더: 체형별 ──
├── 2족 (824)
├── 4족 (414)
└── 기타 (비행, 벌레 등)
```

동작(공격/사망/대기/포효/이동/피격/등장/기절)은 **태그 필터**로 제공. 폴더 2축은 구조적으로 불가능하므로, 체형을 폴더로, 동작을 태그로 분리.

## 탭별 좌측 패널 구성

탭 전환 시 좌측 패널에 노출되는 폴더와 태그 필터가 달라진다.

### 연출 탭

**폴더:**
- 연기 섹션 전체 (대화, 리액션, 생활 동작)
- 연출 섹션 전체 (연출 기법, 게임 연출)
- 전투 섹션 전체 (이동, 교전, 피격, 리턴, 대기) — 공유

**태그 필터:**
- 감정 (8그룹) — **강조 표시**
- 작품
- 상황, 성격, 인물
- 물건, 장소/환경
- 효과

### 게임 탭

**폴더:**
- 전투 섹션 전체 (이동, 교전, 피격, 리턴, 대기) — 공유
- 장비 섹션 전체 (무기별, 포즈)
- 연기 섹션 일부 (대화, 생활 동작)

**태그 필터:**
- 클래스 (6종) — **강조 표시**
- 작품
- 감정 (8그룹)
- 메타 (한손/양손/일대일 등)
- 스킬 특징

### 몬스터 탭

**폴더:**
- 체형 (2족, 4족, 기타)

**태그 필터:**
- 동작 (공격, 사망, 대기, 포효, 이동, 피격, 등장, 기절)
- 종류 (좀비, 악마, 보스, 골렘, 새...)
- 작품

## 필터링 로직

```
폴더 필터: OR (선택된 폴더 중 하나라도 소속이면 표시)
태그 필터: AND (선택된 태그를 모두 포함해야 표시)
탭 필터: 데이터셋 전환 (몬스터) 또는 패널 프리셋 변경 (연출/게임)
별점 필터: >= (선택 별점 이상)
검색: Pagefind 결과 ID ∩ 현재 필터 결과
정렬: 최신순 | 별점순 | 이름순
```

### 자동화 가능 범위

- **교전 분할**: 무기 태그("검", "활", "총", "마법" 등)가 있으면 → 근접/원거리 자동 판별
- **orphan 분류**: 감정 태그("놀람", "슬픔", "화남" 등)가 있으면 → 리액션 자동 분류
- **orphan 분류**: 행동 태그("앉기", "잡기", "일어나기" 등)가 있으면 → 생활 동작 자동 분류
- 나머지는 수동 확인 필요

## 사용자 여정 검증 결과

20개 시나리오 시뮬레이션 (10종 사용자 × 2개 시나리오):

| 결과 | 수 | 비율 |
|---|---|---|
| PASS | 10 | 50% |
| PASS with friction | 4 | 20% |
| PARTIAL PASS | 2 | 10% |
| FAIL | 2 | 10% |
| 구조적 문제 | 1 | 5% |
| 평가 불가 | 1 | 5% |

**FAIL 원인과 대응:**
- 카메라 전환 기법 (28개 과소) → 연출 기법 폴더에 미분류 아이템 재분류 시 해결
