# 비디오 플레이어 리디자인

## 목표

VideoPlayer 컴포넌트의 컨트롤 UI를 Eagle 스타일로 리디자인하고, 애니메이터 워크플로우에 필요한 기능을 추가한다.

## 현재 상태

- 텍스트 버튼 ("Play"/"Pause"), 아이콘 없음
- 프로그레스바가 비디오 아래 분리 배치, gap으로 떨어져 있음
- 기능: 재생/정지, 클릭 seek, 속도 순환 (0.25x~2x, 7단계)
- 프레임 탐색, 음소거 토글, 구간 반복 없음
- `<video loop>` 하드코딩, `muted`는 props로 결정

## 변경 사항

### 1. 컨트롤바 리디자인

**레이아웃:** 비디오 아래 항상 노출되는 컨트롤바. 호버 표시/숨김 없음. 비디오와 컨트롤바 사이 gap 없이 밀착.

**배치 (왼→오):**

| 위치 | 요소 | 동작 |
|------|------|------|
| 1 | ▶/⏸ 재생/정지 아이콘 | 클릭으로 토글 |
| 2 | 🔇/🔊 음소거 아이콘 | 클릭으로 음소거 토글 |
| 3 | `0:01 / 0:08` 시간 표시 | 현재시간 / 총시간 (M:SS 형식) |
| 4 | `F:15` 프레임 카운터 | 현재 프레임 번호 (15fps 고정) |
| 5 | 시크바 | flex 확장, 드래그 핸들 + In/Out 마커 |
| 6 | `1x` 속도 | 클릭으로 순환 (0.25x → 0.5x → 0.75x → 1x → 1.25x → 1.5x → 2x) |
| 7 | 🔁 루프 아이콘 | 반복 재생 on/off, 활성 시 accent 색상 |

**스타일:**
- 모든 아이콘: SVG, 16px 통일
- 텍스트 라벨 없음, 아이콘만 (각 버튼에 `aria-label` 필수)
- 다크/라이트 테마 대응 (기존 CSS 변수 활용)
- 시크바: 2px 높이 트랙 + 10px 원형 핸들
- 컨트롤바 배경: `bg-surface`, 비디오와 `border-border`로 구분

### 2. 시크바 스크러빙

- 핸들 드래그 시작 → 재생 일시정지
- 드래그 중 → `video.currentTime` 업데이트로 프레임 탐색
- 드래그 놓으면 → 해당 위치에서 정지 상태 유지
- 트랙 클릭 → 해당 위치로 즉시 이동 (기존 동작 유지)
- 드래그: `pointerdown` → `pointermove` → `pointerup` 패턴 (마우스 + 터치 통합)

### 3. In/Out 마커 (재생 범위)

- 시크바에 In/Out 마커 2개가 **항상 표시**
- 기본 위치: In = 0:00, Out = 총 시간 (전체 구간)
- 마커를 드래그하여 재생 범위 조정
- 재생 시 Out 지점에 도달하면:
  - 루프 on → In 지점으로 돌아가서 반복
  - 루프 off → Out 지점에서 정지
- 시크바에 In~Out 구간을 반투명 하이라이트로 표시
- `X` 키로 마커 초기화 (전체 구간으로 복원)

**시각적 표현:**
- In/Out 마커: accent 색상의 세로 바 (2px 너비, 14px 높이)
- 구간 하이라이트: accent 색상 30% 투명도
- 마커 드래그 시 커서 변경 (`col-resize`)

**드래그 대상 우선순위:** 시크바에 3개의 드래그 대상(핸들, In 마커, Out 마커)이 공존. 핸들과 마커가 겹칠 때는 핸들 우선. 마커 간 최소 간격 없음 (같은 위치 허용 = 구간 0).

### 4. 프레임 카운터

- 시간 표시 옆에 `F:0` 형식으로 현재 프레임 번호 표시
- 계산: `Math.floor(currentTime * 15)` (15fps 고정, 현재 export 파이프라인이 15fps 단일 스펙)
- 색상: 시간 표시보다 약간 더 뮤트된 색상

### 5. 키보드 단축키

| 키 | 동작 | 적용 범위 |
|----|------|----------|
| `Space` | 재생 / 정지 | ClipDetailView, QuickViewModal |
| `←` | 1초 뒤로 | ClipDetailView, QuickViewModal |
| `→` | 1초 앞으로 | ClipDetailView, QuickViewModal |
| `M` | 음소거 토글 | ClipDetailView, QuickViewModal |
| `X` | In/Out 마커 초기화 | ClipDetailView, QuickViewModal |
| `Escape` | 모달 닫기 | QuickViewModal만 |
| `+`/`=` | 속도 올리기 | QuickViewModal (기존 유지) |
| `-` | 속도 내리기 | QuickViewModal (기존 유지) |

**QuickViewModal 키보드 변경:**
- `Space`: 모달 닫기 → 재생/정지로 변경
- `←`/`→`: 이전/다음 클립 탐색 → 1초 뒤로/앞으로 변경
- 모달 닫기는 `Escape`만으로 통일

### 6. 속도 옵션

현재 코드의 7단계를 그대로 유지: `[0.25, 0.5, 0.75, 1, 1.25, 1.5, 2]`

QuickViewModal의 `+`/`-` 키 속도 조절도 기존 동작 유지.

### 7. 루프 동작 변경

- 기존: `<video loop>` 하드코딩으로 항상 반복
- 변경: `loop` attribute 제거, JavaScript로 루프 로직 직접 제어
- `timeupdate` 이벤트에서 Out 마커 도달 감지 → 루프 상태에 따라 In 지점 복귀 또는 정지
- 루프 토글 상태는 VideoPlayer 내부 state, 기본값: on (기존 동작 유지)

### 8. 음소거 동작 변경

- 기존: `muted` prop으로만 결정 (`compact || autoPlayMuted`)
- 변경: `muted`를 내부 state로 관리, 초기값만 prop에서 결정
- 사용자가 버튼/M키로 언제든 토글 가능

## 적용 대상

| 요소 | compact=false (상세) | compact=true (퀵뷰) |
|------|---------------------|---------------------|
| 재생/정지 | O | O |
| 음소거 | O (기본 off) | O (기본 on, 토글 가능) |
| 시간 표시 | O | O |
| 프레임 카운터 | O | O |
| 시크바 + In/Out 마커 | O | O |
| 속도 | O (클릭 순환) | O (+/- 키로 조절) |
| 루프 | O | O |

## 기술 노트

- H.264 MP4에서 `currentTime` 설정 시 가장 가까운 키프레임으로 snap될 수 있으나, export 파이프라인이 CRF 28 / 15fps로 인코딩하므로 키프레임 간격이 짧아 실용적으로 충분히 정밀
- In/Out 마커, 루프, 음소거 상태는 VideoPlayer 내부 state (URL 동기화 불필요)
- 에러 발생 시 (`hasPlaybackError`): 모든 컨트롤 disabled, 기존 "Unavailable" 표시 유지

## 비적용 사항

- 볼륨 슬라이더 (음소거 토글만으로 충분)
- 풀스크린 버튼 (보안상 다운로드 방지 유지)
- 프레임 캡처/스크린샷
- 이전/다음 프레임 버튼 (스크러빙으로 대체)
- 시크바 호버 시간 프리뷰 (추후 고려)
