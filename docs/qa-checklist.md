# Reflix QA 체크리스트 (일반 사용자 기준, 2026-03-29)

> 이 문서는 "개발자가 아니라 실제 사용자" 시선으로 손으로 눌러보며 확인하는 QA 목록입니다.  
> 내부 구현보다 "화면이 자연스러운지, 막히지 않는지, 기대한 결과가 나오는지"를 봅니다.

## 테스트 전 준비

- [ ] 데스크톱 1종(Chrome 권장)과 모바일 1종(Safari 또는 Chrome)에서 확인한다.
- [ ] 아래 3가지 상태를 준비한다: `게스트(로그아웃)`, `Free(로그인, 미구독)`, `Pro(로그인, 구독 중)`.
- [ ] 각 시나리오를 시작하기 전에 새 탭에서 다시 열어 이전 검색/필터 영향이 없는지 확인한다.
- [ ] 버그를 발견하면 `계정 상태 / 페이지 / 재현 순서 / 실제 결과 / 기대 결과 / 스크린샷`을 함께 남긴다.

## 현재 정책 빠른 요약

- 게스트는 둘러보기와 상세 페이지 접근이 가능하고, 공유된 상세 링크에서는 원본 영상도 바로 재생할 수 있어야 한다.
- 게스트는 기본 browse에서 실제로 열 수 있는 teaser 결과가 3개까지여야 한다.
- 게스트가 검색이나 필터를 쓰면 결과는 계속 보이되, 썸네일이 블러되고 로그인 전에는 열리지 않아야 한다.
- Free는 로그인 후 원본 영상 재생이 가능하고, browse에서 실제로 열 수 있는 결과는 5개까지여야 한다.
- Free는 보드 1개까지만 만들 수 있어야 한다.
- 게스트와 Free는 필터를 2축 이상 동시에 깊게 쓰는 흐름이 그대로 허용되면 안 된다.
- Pro는 잠금 없이 전체 결과, 다중 필터, 셔플, 보드 무제한을 써야 한다.

## 1. 첫 방문 경험: 랜딩 (`/ko`)

- [ ] 첫 화면이 깨짐 없이 열리고, 상단 네비게이션, 히어로, 기능 소개, 통계, 요금제 순서가 자연스럽게 보인다.
- [ ] 상단 로고를 누르면 항상 `/ko` 홈으로 돌아온다.
- [ ] 상단 `탐색하기` 또는 메인 CTA를 누르면 `/ko/browse`로 이동한다.
- [ ] 상단 `요금제`를 누르면 같은 페이지의 요금제 섹션으로 자연스럽게 스크롤된다.
- [ ] 언어 전환을 누르면 `/en`으로 바뀌고, 다시 돌아와도 레이아웃이 깨지지 않는다.
- [ ] 모바일에서는 햄버거 메뉴가 열리고, 메뉴 안의 `탐색`, `요금제`, 언어 전환이 모두 동작한다.
- [ ] 요금제 섹션에서 `월간 / 연간` 전환 시 Pro 가격과 할인 안내가 함께 바뀐다.
- [ ] Free CTA는 browse로 이동하고, Pro CTA는 가격 모달을 연다.

## 2. 게스트 흐름: 로그인하지 않고 둘러보기 (`/ko/browse`)

- [ ] 로그인 없이 browse 페이지에 들어갈 수 있다.
- [ ] 첫 진입 시 브랜드 스플래시가 잠깐 나오더라도 멈추지 않고 본 화면으로 넘어간다.
- [ ] 필터가 없을 때는 피드형 섹션 뷰가 자연스럽고, 검색, 태그, 폴더를 걸면 결과 중심 그리드로 바뀐다.
- [ ] 검색창에 단어를 입력하면 결과가 바로 바뀌고, 최근 검색어, 추천 태그, 태그 제안이 어색하지 않다.
- [ ] 언어를 KO ↔ EN으로 바꿔도 현재 페이지와 검색 조건이 최대한 유지된다.
- [ ] 좌우 패널 접기/펼치기 버튼이 정상 동작하고, 콘텐츠 영역이 깨지지 않는다.
- [ ] 좌측의 `모든 태그`, 폴더 트리, `최근 본 항목` 메뉴가 각각 정상 동작한다.
- [ ] 결과가 많은 상태에서 스크롤해도 카드 썸네일과 미리보기가 부드럽게 로딩된다.
- [ ] 게스트 기본 browse에서는 실제로 열 수 있는 teaser 결과가 3개까지만 보장된다.
- [ ] 게스트가 검색, 태그, 폴더 같은 조건을 쓰면 매칭 결과가 사라지지 않고 계속 보이되 썸네일이 블러 처리된다.
- [ ] 블러된 카드 클릭 시 카드가 열리지 않고 `/login` 페이지와 같은 로그인 UI 모달이 떠야 한다.
- [ ] 검색/필터 결과 상단에는 `로그인하면 결과를 열 수 있어요` 같은 로그인 유도 문구가 보인다.
- [ ] 게스트 상태에서 여러 필터를 동시에 과하게 걸어도 결과가 깨지지 않고, 블러/로그인 게이트 또는 제한된 동작으로 정리된다.
- [ ] 카드 우측 상단 저장 버튼을 누르면 보드 저장 팝오버 대신 로그인 또는 업그레이드 흐름으로 이어진다.
- [ ] 상세 페이지 URL로 직접 들어가면 게스트도 원본 영상이 바로 재생되어야 한다.
- [ ] AI 분석이 있는 클립은 게스트도 설명 텍스트를 볼 수 있어야 한다.

## 3. 로그인 흐름 (`/ko/login`)

- [ ] 이메일 입력 후 `매직 링크 보내기`를 누르면 로딩 상태가 보이고 중복 클릭이 막힌다.
- [ ] 전송 성공 후 `이메일을 확인하세요` 안내 화면으로 바뀐다.
- [ ] 다른 탭이나 메일 앱에서 로그인해도 원래 탭이 자동으로 다음 화면으로 이동한다.
- [ ] 너무 빠르게 반복 요청하거나 잘못된 이메일을 넣었을 때 에러 문구가 이해 가능하게 보인다.
- [ ] `Google로 계속하기`를 누르면 Google 인증 흐름으로 이동하고, 완료 후 앱으로 돌아온다.
- [ ] 게스트가 블러된 카드에서 로그인한 경우, 로그인 후 원래 browse 검색/필터 상태로 돌아온다.
- [ ] 로그인 전에 누른 클립이 Free 범위 안이면, 복귀 후 그 클립이 자동 선택되거나 바로 열려 흐름이 끊기지 않는다.
- [ ] 로그인 후에도 그 클립이 Free 범위를 넘으면, 앱이 멈추지 않고 다시 Pro 업그레이드 유도로 이어진다.

## 4. Free 사용자 흐름

- [ ] 로그인 후 상단 `로그인` 버튼이 사용자 메뉴 또는 계정 진입으로 바뀐다.
- [ ] browse에서 실제로 열 수 있는 결과는 5개까지여야 하고, 그 이후 결과는 잠겨 있어야 한다.
- [ ] Free 사용자가 잠긴 결과를 누르면 로그인 CTA가 아니라 Pro 업그레이드 중심 모달이 떠야 한다.
- [ ] Free에서는 원본 영상 재생이 정상 동작한다.
- [ ] 검색과 태그 탐색은 가능하지만, `검색어 + 태그`, `카테고리 + 태그` 같은 다중 필터 조합은 그대로 무제한 허용되지 않아야 한다.
- [ ] 다중 필터를 시도했을 때 결과가 깨지지 않고, Pro 안내가 보이거나 1축 기준으로 안정적으로 동작해야 한다.
- [ ] 카드 저장 버튼으로 보드에 저장하거나 해제할 수 있다.
- [ ] Free 계정은 보드를 1개까지 만들 수 있고, 2개째부터는 분명한 제한 문구가 보여야 한다.
- [ ] `내 보드`에서 보드 생성, 이름 변경, 삭제, 보드 안 결과 보기 흐름이 모두 정상이다.
- [ ] `최근 본 항목`에 방금 본 클립이 쌓이고, `기록 지우기`가 동작한다.

## 5. Pro 사용자 흐름

- [ ] browse 결과에 잠긴 카드가 없어야 한다.
- [ ] 상단 셔플 버튼이 보이고 누를 때 결과 순서가 실제로 바뀐다.
- [ ] 검색, 태그, 폴더 같은 다중 필터 조합이 동시에 정상 적용된다.
- [ ] 카드 저장, 보드 생성, 보드 여러 개 운영이 제한 없이 동작한다.
- [ ] 유료 Pro 사용자가 가격 모달을 열어도 중복 결제로 헷갈리게 하지 않는다.
- [ ] Pro 상태에서 QuickView, 상세 페이지, 공유, 계정 페이지 이동까지 전체 흐름이 막힘 없이 이어진다.

## 6. 클립 보기 경험: 카드, QuickView, 상세 페이지

- [ ] 카드 1회 클릭 시 선택 상태가 표시되고 우측 정보 패널이 바뀐다.
- [ ] 카드 2회 클릭 시 QuickView가 열린다.
- [ ] QuickView는 배경 클릭이나 `ESC`로 닫을 수 있다.
- [ ] QuickView 안 `상세 보기` 버튼을 누르면 해당 클립의 상세 페이지로 이동한다.
- [ ] 공유 버튼을 누르면 현재 클립 상세 URL이 복사되고, 복사 완료 문구가 잠시 보여야 한다.
- [ ] QuickView 안의 관련 클립 썸네일을 누르면 다른 클립으로 자연스럽게 넘어간다.
- [ ] 상세 페이지 자체는 URL로 직접 접근 가능하고, 제목, 설명, 태그, 폴더, 속성 정보가 읽기 쉽게 보인다.
- [ ] 공유된 상세 링크를 게스트로 열어도 원본 영상 재생이 막히지 않는다.
- [ ] AI 분석 섹션이 있는 클립은 펼치기/접기가 자연스럽고, 내용이 비어 있거나 깨지지 않는다.

## 7. 비디오 플레이어

- [ ] 클릭으로 재생/일시정지가 된다.
- [ ] 시크바 드래그로 원하는 지점으로 이동할 수 있다.
- [ ] 음소거, 속도 변경, 반복, 전체화면, 확장 버튼이 모두 동작한다.
- [ ] 시간 표시와 프레임 표시 전환이 된다.
- [ ] In/Out 포인트를 잡고 반복 구간 재생이 된다.
- [ ] 키보드 단축키 `Space`, `←`, `→`, `,`, `.`, `m`, `l`, `i`, `o`, `x`, `f`, `e`, `-`, `+`가 기대한 대로 동작한다.
- [ ] 플레이어를 크게 확장해도 레이아웃이 깨지지 않고, 다시 축소하면 정보 패널이 돌아온다.
- [ ] 재생 실패가 나더라도 화면 전체가 멈추거나 버튼이 먹통이 되지 않는다.

## 8. 계정 페이지 (`/ko/account`)

- [ ] 로그아웃 상태로 `/ko/account`에 들어가면 `/ko/login`으로 이동한다.
- [ ] 로그인 상태에서는 `프로필 / 구독 / 보안` 탭이 보이고 탭 이동이 자연스럽다.
- [ ] 프로필 탭에서 표시 이름을 수정하고 저장할 수 있다.
- [ ] 프로필 탭에서 현재 이메일이 정확히 보인다.
- [ ] 구독 탭에서 현재 상태가 `FREE`, `PRO`, `베타` 중 맞게 표시된다.
- [ ] Free 사용자는 `Pro로 업그레이드` 버튼으로 가격 모달을 열 수 있다.
- [ ] 유료 Pro 사용자는 `구독 관리` 버튼이 보이고 동작 중 로딩 상태가 보인다.
- [ ] 보안 탭에서 Google 연결 상태가 정확히 보인다.
- [ ] 아직 연결되지 않았다면 Google 연결 버튼으로 연동을 시작할 수 있다.
- [ ] 이메일 변경 요청을 보내면 성공 또는 실패 메시지가 이해 가능하게 보인다.
- [ ] 로그아웃 버튼을 누르면 홈으로 돌아가고, 다시 보호된 페이지에 바로 들어가지 못한다.
- [ ] 계정 삭제 확인 흐름은 오작동 없이 매우 신중하게 동작해야 하며, 확인 문구가 정확히 맞을 때만 삭제 버튼이 가능해져야 한다.

## 9. 모바일, 언어, 테마

- [ ] 모바일 browse에서 검색 아이콘을 누르면 전체 화면 검색 오버레이가 열린다.
- [ ] 모바일 검색 오버레이는 닫기 버튼으로 닫히고, 결과 카드 선택 후 원래 화면 흐름이 이어진다.
- [ ] 모바일에서 패널 토글 후에도 콘텐츠가 잘리거나 겹치지 않는다.
- [ ] QuickView와 가격 모달이 모바일에서 화면 밖으로 잘리지 않고 세로 스크롤이 가능하다.
- [ ] 라이트/다크 테마 전환이 즉시 반영되고 새로고침 후에도 유지된다.
- [ ] 랜딩 페이지는 항상 다크 톤으로 보이고, browse와 account는 선택한 테마를 따른다.
- [ ] KO ↔ EN 전환 시 현재 페이지와 주요 상태(URL, 검색어)가 최대한 유지된다.
- [ ] 영어 화면에서도 핵심 버튼, 라벨, 빈 상태 문구가 자연스럽고 잘리지 않는다.

## 10. 복구/회귀 체크

- [ ] 검색 결과가 0개일 때 이해 가능한 빈 상태 문구가 보인다.
- [ ] 없는 클립 URL이나 잘못된 페이지 주소에 들어가면 깨진 화면 대신 정상적인 오류 또는 404 흐름으로 처리된다.
- [ ] 브라우저 뒤로가기와 앞으로가기를 눌렀을 때 검색, 필터, 보드 상태가 크게 어긋나지 않는다.
- [ ] 페이지 새로고침 후에도 URL에 들어 있던 검색, 필터 조건이 복원된다.
- [ ] 보드 화면, 최근 본 항목, 태그 화면 사이를 오가도 화면이 멈추지 않는다.
- [ ] 느린 네트워크에서도 썸네일이나 비디오가 영원히 빈칸으로 남지 않는다.
- [ ] 눈에 보이는 수준의 레이아웃 깨짐, 무한 로딩, 버튼 무반응, 잘못된 언어 혼합이 없다.
