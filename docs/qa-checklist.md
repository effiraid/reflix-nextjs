# Reflix QA 체크리스트 (현재 코드 기준, 2026-03-29)

> 이 문서는 "기획 의도"가 아니라 "지금 저장소의 실제 코드"를 기준으로 적은 QA 목록입니다.  
> 문구나 정책이 예전 자료와 다르더라도, 이 문서에서는 현재 구현된 동작을 우선해서 확인합니다.

## 테스트 전 준비

- [ ] 데스크톱 1종(Chrome 권장)과 모바일 1종(Safari 또는 Chrome)에서 확인한다.
- [ ] 아래 4가지 상태를 준비한다: `게스트(로그아웃)`, `Free(로그인, 미구독)`, `Pro(로그인, 유료 구독)`, `Beta(Pro 체험 중)`.
- [ ] 같은 시나리오를 다시 볼 때는 새 탭에서 열거나 새로고침해서 이전 검색/필터/보드 상태가 섞이지 않는지 본다.
- [ ] 버그를 발견하면 `계정 상태 / 페이지 / 재현 순서 / 실제 결과 / 기대 결과 / 스크린샷`을 같이 남긴다.

## 현재 정책 빠른 요약

- 게스트도 랜딩, browse, 상세 페이지에 들어갈 수 있다.
- 게스트도 공유된 상세 링크(`/clip/{id}`)에서는 원본 영상을 바로 재생할 수 있다.
- browse/search에서는 게스트, Free, Pro 모두 결과 목록 자체는 볼 수 있다.
- 게스트는 browse/search에서 카드가 전부 잠겨 있어야 하고, 어떤 카드도 열 수 없어야 한다.
- 게스트가 잠긴 카드, `내 보드`, `최근 기록`을 누르면 별도 페이지 이동보다 "공용 로그인 모달"이 열려야 한다.
- Free는 browse/search 결과 전체를 보되, 앞의 5개 카드만 열 수 있어야 한다.
- Free는 보드 1개까지만 만들 수 있어야 한다.
- Free와 게스트는 필터를 2축 이상 조합해도 그대로 전부 적용되지 않고, 1축 기준으로 안정적으로 제한되어야 한다.
- Pro와 Beta(Pro 체험)는 잠금 없이 전체 결과, 다중 필터, 셔플, 보드 무제한을 써야 한다.
- AI 분석 텍스트는 현재 코드상 Pro 전용이 아니므로, 게스트/Free도 상세에서 볼 수 있어야 한다.

## 1. 첫 방문 경험: 랜딩 (`/ko`)

- [ ] 첫 화면이 깨짐 없이 열리고, 상단 네비게이션, 히어로, 기능 소개, 통계, 요금제, 마지막 CTA 순서가 자연스럽게 보인다.
- [ ] 상단 로고를 누르면 항상 현재 언어 홈(`/ko` 또는 `/en`)으로 돌아간다.
- [ ] 상단 `탐색`, 히어로 CTA, 마지막 CTA를 누르면 `/ko/browse`로 이동한다.
- [ ] 상단 `요금제`를 누르면 같은 페이지의 요금제 섹션으로 부드럽게 스크롤된다.
- [ ] 언어 전환을 누르면 `/en` 또는 `/ko`로 바뀌고, 레이아웃이 깨지지 않는다.
- [ ] 모바일에서는 햄버거 메뉴가 바텀시트처럼 열리고, 그 안의 `탐색`, `요금제`, 언어 전환, CTA가 모두 동작한다.
- [ ] 요금제 섹션에서 `월간 / 연간` 전환 시 Pro 가격과 하단 과금 안내 문구가 함께 바뀐다.
- [ ] 요금제 섹션의 Free CTA는 browse로 이동하고, Pro CTA는 가격 모달을 연다.
- [ ] 랜딩 전체는 다크 톤으로 고정되어 보이고, 중간에 라이트 테마처럼 섞여 보이지 않는다.

## 2. 게스트 흐름: 로그인하지 않고 둘러보기 (`/ko/browse`)

- [ ] 로그인 없이 browse 페이지에 들어갈 수 있다.
- [ ] 첫 진입 시 브랜드 스플래시가 잠깐 보여도 멈추지 않고 실제 화면으로 넘어간다.
- [ ] 필터가 없을 때는 카테고리별 피드형 화면이 보이고, `전체 보기`나 검색/태그/폴더를 쓰면 결과 중심 그리드로 전환된다.
- [ ] 검색창에 단어를 입력하면 URL의 `q`와 결과가 함께 바뀌고, 최근 검색어, 추천 태그, 태그 제안이 어색하지 않다.
- [ ] 언어를 KO ↔ EN으로 바꿔도 현재 browse 경로와 검색어 쿼리가 최대한 유지된다.
- [ ] 상단 패널 토글이 정상 동작하고, 좌우 패널을 접거나 펼쳐도 본문이 깨지지 않는다.
- [ ] 좌측 메뉴의 `둘러보기`, `모든 태그`, `내 보드`, `최근 기록`, 폴더 트리가 각각 눌린다.
- [ ] `내 보드`나 `최근 기록`을 게스트 상태에서 누르면 별도 빈 페이지가 아니라 로그인 모달이 떠야 한다.
- [ ] 결과 카드의 썸네일/프리뷰는 보이더라도, browse 안에서는 모든 카드가 잠금 상태여야 한다.
- [ ] 게스트는 browse/search에서 "열 수 있는 teaser 3개"가 아니라 "0개 열람 가능" 상태여야 한다.
- [ ] 검색, 태그, 폴더 같은 조건을 걸면 결과는 계속 보이되 카드 미디어가 블러되고 잠금 오버레이가 보여야 한다.
- [ ] 잠긴 카드 클릭 또는 더블클릭 시 상세/퀵뷰로 가지 않고 `/login`과 같은 로그인 UI가 모달로 떠야 한다.
- [ ] 검색/필터 결과 상단에는 `로그인하면 결과를 열 수 있어요`에 해당하는 안내가 보여야 한다.
- [ ] 카드 우측 상단 저장 버튼을 누르면 보드 저장 팝오버가 아니라 로그인 모달이 떠야 한다.
- [ ] 여러 필터를 동시에 걸어도 화면이 깨지지 않고, 실제 적용은 1축 중심으로 제한되어야 한다.
- [ ] 상세 페이지 URL로 직접 들어가면 게스트도 원본 영상이 재생되어야 한다.
- [ ] AI 분석이 있는 클립은 게스트도 설명 텍스트를 볼 수 있어야 한다.

## 3. 로그인 흐름 (`/ko/login` + 공용 로그인 모달)

- [ ] 이메일 입력 후 `매직 링크 보내기`를 누르면 로딩 상태가 보이고 중복 클릭이 막힌다.
- [ ] 전송 성공 후 `이메일을 확인하세요` 안내 화면으로 바뀐다.
- [ ] 다른 탭이나 메일 앱에서 로그인하면 원래 탭이 자동으로 다음 화면으로 이동한다.
- [ ] 너무 빠르게 반복 요청했을 때는 `잠시 후 다시 시도해주세요 (60초 제한)` 수준의 이해 가능한 안내가 보여야 한다.
- [ ] 리다이렉트 설정이나 로그인 설정이 잘못된 경우에도 완전히 멈추지 않고 에러 문구가 보여야 한다.
- [ ] `Google로 계속하기`를 누르면 Google 인증 흐름으로 이동하고, 완료 후 앱으로 돌아온다.
- [ ] 잠긴 카드에서 로그인한 경우, 로그인 후 원래 browse URL과 검색/필터 상태로 최대한 복귀해야 한다.
- [ ] 로그인 전에 누른 클립이 Free 범위 안이면 복귀 후 그 클립이 다시 선택되거나 퀵뷰가 열려 흐름이 이어져야 한다.
- [ ] 로그인 후에도 그 클립이 Free 범위를 넘는 잠금 카드면, 앱이 멈추지 않고 가격 모달로 이어져야 한다.
- [ ] `내 보드` 또는 `최근 기록`에서 뜬 로그인 모달도 같은 로그인 카드 UI를 재사용해야 한다.

## 4. Free 사용자 흐름

- [ ] 로그인 후 상단 `로그인` 버튼이 사용자 메뉴로 바뀐다.
- [ ] browse/search 결과는 전체가 보이지만, 실제로 열 수 있는 카드는 앞 5개까지만 허용되어야 한다.
- [ ] Free 사용자가 6번째 이후 잠긴 카드를 누르면 로그인 모달이 아니라 가격 모달이 떠야 한다.
- [ ] Free는 browse에서도 원본 영상 재생이 가능해야 한다.
- [ ] 검색, 폴더, 태그 중 한 축은 정상적으로 쓸 수 있어야 한다.
- [ ] `검색어 + 태그`, `폴더 + 태그`처럼 2축 이상을 동시에 쓰면 결과가 깨지지 않고, Pro 안내와 함께 1축 기준으로 안정적으로 제한되어야 한다.
- [ ] 카드 저장 버튼을 누르면 보드 저장 팝오버가 열리고, 보드에 저장/해제가 된다.
- [ ] Free 계정은 보드를 1개까지만 만들 수 있고, 2개째부터는 새 보드 생성이 막히고 제한 안내가 보여야 한다.
- [ ] 이미 있는 보드는 계속 열 수 있고, 이름 변경/삭제도 정상 동작해야 한다.
- [ ] `내 보드`에서 보드를 눌렀을 때 해당 보드 필터가 적용된 browse 화면으로 자연스럽게 들어가야 한다.
- [ ] `최근 기록`에 방금 본 클립이 쌓이고, 개별 삭제와 `기록 삭제`가 둘 다 동작해야 한다.

## 5. Pro / Beta(Pro 체험) 사용자 흐름

- [ ] Pro 또는 Beta 상태에서는 browse/search 결과에 잠긴 카드가 없어야 한다.
- [ ] 그리드 툴바에서 셔플 버튼이 보이고, 눌렀을 때 실제 카드 순서가 바뀐다.
- [ ] 검색, 태그, 폴더 같은 다중 필터 조합이 동시에 정상 적용된다.
- [ ] 보드 생성/저장/해제가 제한 없이 동작한다.
- [ ] 유료 Pro는 계정/메뉴에서 `PRO` 상태가 자연스럽게 보이고, 결제 중복처럼 헷갈리는 문구가 없어야 한다.
- [ ] Beta(Pro 체험)는 계정 화면에서 `BETA` 상태가 보일 수 있고, 실제 browse 권한은 Pro처럼 동작해야 한다.
- [ ] Pro/Beta 상태에서 QuickView, 상세 페이지, 공유, 계정 이동 흐름이 끊기지 않아야 한다.

## 6. 클립 보기 경험: 카드, 우측 패널, QuickView, 상세 페이지

- [ ] 카드 1회 클릭 시 선택 상태가 표시되고 우측 정보 패널이 해당 클립 내용으로 바뀐다.
- [ ] 같은 카드를 다시 클릭하거나 더블클릭하면 QuickView가 열린다.
- [ ] QuickView는 배경 클릭이나 `ESC`로 닫을 수 있다.
- [ ] QuickView 안 `상세 보기` 버튼을 누르면 해당 클립의 상세 페이지로 이동한다.
- [ ] QuickView 안의 관련 클립을 누르면 다른 클립으로 자연스럽게 바뀐다.
- [ ] 공유 버튼을 누르면 현재 클립 상세 URL이 복사되고, 잠시 `복사됨` 상태가 보여야 한다.
- [ ] 상세 페이지는 URL로 직접 접근 가능하고, 제목, 설명, 태그, 폴더, 속성 정보가 읽기 쉽게 보여야 한다.
- [ ] 공유된 상세 링크를 게스트로 열어도 원본 영상 재생이 막히지 않아야 한다.
- [ ] AI 분석이 있는 클립은 섹션이 비어 있거나 깨지지 않고, 펼치기/접기가 자연스럽게 동작해야 한다.

## 7. 비디오 플레이어

- [ ] 비디오 영역 클릭으로 재생/일시정지가 된다.
- [ ] 시크바 드래그로 원하는 지점으로 이동할 수 있다.
- [ ] 비디오 영역 드래그 스크럽도 동작하고, 드래그 중에는 자동으로 재생되지 않아야 한다.
- [ ] 음소거, 속도 변경, 반복, 전체화면, 확장 버튼이 모두 동작한다.
- [ ] 시간 표시와 프레임 표시 전환이 된다.
- [ ] In/Out 포인트를 잡고 반복 구간 재생이 된다.
- [ ] 키보드 단축키 `Space`, `←`, `→`, `,`, `.`, `m`, `l`, `i`, `o`, `x`, `f`, `e`, `-`, `+`가 기대대로 동작한다.
- [ ] `[`와 `]`도 In/Out 포인트 보조 단축키로 동작한다.
- [ ] 플레이어를 크게 확장해도 레이아웃이 깨지지 않고, 다시 축소하면 옆 정보 패널이 돌아온다.
- [ ] 재생 실패가 나더라도 화면 전체가 멈추거나 버튼이 전부 먹통이 되지 않는다.

## 8. 계정 페이지 (`/ko/account`)

- [ ] 로그아웃 상태로 `/ko/account`에 들어가면 `/ko/login`으로 이동한다.
- [ ] 로그인 상태에서는 `프로필 / 구독 / 보안` 탭이 보이고, 탭 이동이 자연스럽다.
- [ ] 프로필 탭에서 닉네임을 수정하고 저장할 수 있다.
- [ ] 프로필 탭에서 현재 이메일이 정확히 보인다.
- [ ] 구독 탭에서 현재 상태가 `FREE`, `PRO`, `BETA` 중 맞게 표시된다.
- [ ] Free 사용자는 `Pro로 업그레이드` 버튼으로 가격 모달을 열 수 있다.
- [ ] 유료 Pro 사용자는 `구독 관리` 버튼이 보이고, 누르면 로딩 상태가 보인다.
- [ ] Beta 사용자는 종료일이 있으면 안내가 보이고, 종료 후 Free로 돌아간다는 설명이 보인다.
- [ ] 보안 탭에서 이메일 로그인 방식은 기본적으로 `연결됨`으로 보인다.
- [ ] Google 연결 상태가 정확히 보이고, 아직 연결되지 않았다면 버튼으로 연동을 시작할 수 있다.
- [ ] 이메일 변경 요청을 보내면 성공 또는 실패 메시지가 이해 가능하게 보인다.
- [ ] 로그아웃 버튼을 누르면 홈으로 돌아간다.
- [ ] 계정 삭제는 확인 모달에서 정확한 문구(`계정 삭제`)를 입력했을 때만 버튼이 활성화되어야 한다.

## 9. 모바일, 언어, 테마

- [ ] 모바일 browse에서 검색 아이콘을 누르면 전체 화면 검색 오버레이가 열린다.
- [ ] 모바일 검색 오버레이는 닫기 버튼과 `ESC`로 닫힌다.
- [ ] 모바일 검색 오버레이에서 결과를 고르면 오버레이가 닫히고, browse 쿼리와 선택 상태가 갱신된다.
- [ ] 모바일에서 좌우 패널 상태가 바뀌어도 콘텐츠가 잘리거나 겹치지 않는다.
- [ ] QuickView와 가격 모달은 모바일에서 화면 밖으로 잘리지 않고 세로 스크롤이 가능하다.
- [ ] 라이트/다크 테마 전환이 즉시 반영되고, 새로고침 후에도 유지된다.
- [ ] 랜딩 페이지는 항상 다크 톤으로 보이고, browse와 account는 선택한 테마를 따른다.
- [ ] KO ↔ EN 전환 시 현재 페이지와 주요 URL 상태(예: 검색어 쿼리)가 최대한 유지된다.
- [ ] 영어 화면에서도 버튼, 라벨, 빈 상태 문구가 어색하거나 잘리지 않는다.

## 10. 복구/회귀 체크

- [ ] 검색 결과가 0개일 때 이해 가능한 빈 상태 문구가 보인다.
- [ ] 없는 클립 URL이나 잘못된 페이지 주소에 들어가면 깨진 화면 대신 정상적인 404 흐름으로 처리된다.
- [ ] 브라우저 뒤로가기/앞으로가기를 눌렀을 때 검색과 필터 상태가 크게 어긋나지 않는다.
- [ ] 페이지 새로고침 후에도 URL에 있던 검색/필터/보드 상태가 최대한 복원된다.
- [ ] `둘러보기`, `모든 태그`, `내 보드`, `최근 기록` 사이를 오가도 멈추지 않는다.
- [ ] 스플래시, 검색 로딩, 보드 로딩 상태가 영원히 계속되지 않아야 한다.
- [ ] 느린 네트워크에서도 썸네일이나 비디오가 영원히 빈칸으로 남지 않아야 한다.
- [ ] 눈에 보이는 레이아웃 깨짐, 무한 로딩, 버튼 무반응, 잘못된 언어 혼합이 없어야 한다.
