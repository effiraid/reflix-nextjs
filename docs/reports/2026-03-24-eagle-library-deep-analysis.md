## 8. 저장소 파이프라인과의 정합성

문서와 스크립트를 함께 보면 현재 Eagle 라이브러리는 이렇게 쓰인다.

- Eagle은 운영 소스다. 스크립트는 `images/<id>.info/metadata.json`, 실 미디어, 썸네일을 읽어 배치를 만든다.
- 하지만 앱 런타임은 Eagle의 라이브 폴더/태그그룹을 직접 읽지 않고, `src/data/categories.json`, `src/data/tag-groups.json` 같은 정적 파일을 소비한다.
- 즉 Eagle 내부 구조는 운영용 진실 소스이고, 앱 분류 JSON은 배포용 해석 레이어다.
- Phase 2는 폴더 없는 `mp4`만 정리한다. 현재 대상은 정확히 40개다.
- 썸네일은 저장소 문서대로 “파일명은 png, 실체는 animated WebP” 관례를 따른다.
- `reflix:published` 10개와 `config/published-state.json` 10엔트리가 일치해 현재 퍼블리시 상태는 정합하다.

## 9. 우선순위 제안

1. 폴더 없는 `mp4` 40개에 대해 `npm run eagle:phase2:review`를 실행해 이름/태그/폴더 정리를 시작한다.
2. 썸네일 누락 3건을 복구한다.
3. 빈 셸 `.info` 22개의 보존 필요성을 판단하고, 필요 없으면 정리한다.
4. `!` 소스 폴더 179개에 대한 정책을 정한다.
5. 라이브 Eagle 구조와 앱의 정적 taxonomy 간 동기화 전략을 검토한다.
