## Project

Reflix — a game animation reference platform. Eagle-style 3-panel browsing with Pinterest masonry grid, animated MP4 loop previews, and an Eagle-to-R2 export pipeline.

- **Design spec:** `docs/superpowers/specs/2026-03-22-reflix-design.md`
- **Implementation plan:** `docs/superpowers/plans/2026-03-23-reflix-implementation.md`

## Commands

```bash
npm run dev      # Dev server (localhost:3000)
npm run build    # Production build
npm run start    # Serve production build
npm run lint     # ESLint (flat config, eslint.config.mjs)

# Tests (Vitest + jsdom + @testing-library/react)
npx vitest                           # Watch mode
npx vitest run                       # Single run (CI)
npx vitest run src/lib/filter.test.ts  # Single test file

# Export pipeline (Eagle → local JSON/media → optional R2 upload)
npm run export:batch           # Export active release batch, prune stale artifacts
npm run export:batch:dry       # Preview without writing
npm run export:batch:r2        # Export + upload to R2

# Release workflow (scan → review → approve → export → mark-published)
npm run release:scan           # Generate proposal from active batch
npm run release:review         # Tag Eagle items with reflix:review-requested
npm run release:approve        # Promote approved items to release-batch.json
npm run release:mark-published # Record successful publish
```

## Tech Stack

Next.js 16, React 19, TypeScript 5, Tailwind CSS 4, Zustand, @tanstack/react-virtual, next-themes, Pagefind, Cloudflare R2 (@aws-sdk/client-s3)

## Next.js 16 Breaking Changes (CRITICAL)

These differ from your training data. **Read `node_modules/next/dist/docs/` before writing route files.**

- `params` and `searchParams` are **Promises** — always `await` them
- Middleware is `proxy.ts` (NOT `middleware.ts`)
- ISR uses `'use cache'` + `cacheLife()` — NOT the old `revalidate` export
- `cacheComponents` is a **top-level** next.config.ts key (NOT under `experimental`)
- Route type helpers like `PageProps<'/[lang]'>` are auto-generated globally
- PWA is built-in: `app/manifest.ts` + `public/sw.js` (NOT next-pwa)
- i18n uses built-in dictionary pattern with `[lang]` dynamic segment (NOT next-intl)

## Environment Variables

- `NEXT_PUBLIC_MEDIA_URL` — leave unset for local/preview; set to `https://media.reflix.dev` for production
- `EAGLE_LIBRARY_PATH` — path to Eagle `.library` folder (default: local desktop path)
- `R2_ACCOUNT_ID`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_BUCKET_NAME` — Cloudflare R2 credentials (only needed for `--r2` uploads)

## Architecture

- **Rendering:** SSG for browse/home/search pages, ISR (`'use cache'`) for clip detail pages
- **Data:** Static JSON in `src/data/` — `index.json` (clip index), `categories.json`, `tag-groups.json`, `tag-i18n.json`, and individual `clips/{id}.json`. Generated by export pipeline, not hand-edited.
- **Export pipeline:** `scripts/export.mjs` reads Eagle library → generates LQIP/thumbnails/previews via FFmpeg → writes JSON to `src/data/` and media to `public/{videos,previews,thumbnails}/` → optionally uploads to R2. Active batch defined in `config/release-batch.json`; publish history in `config/published-state.json`.
- **i18n:** `[lang]` route segment (`/ko/`, `/en/`), `proxy.ts` handles locale detection + redirect, dictionaries in `src/app/[lang]/dictionaries/`
- **State:** Zustand stores (filterStore, clipStore, uiStore) with unidirectional URL sync (URL = source of truth)
- **Media:** 3-stage loading: LQIP base64 (~300B) → Static WebP thumbnail (~4KB) → MP4 loop preview (~70KB). 상세: `docs/media-strategy.md`
- **Path alias:** `@/*` maps to `./src/*`

## Conventions

- Korean is the primary language for UI strings and data; English as secondary
- Component filenames: PascalCase (e.g., `ClipCard.tsx`)
- Stores: camelCase with `Store` suffix (e.g., `filterStore.ts`)
- Server components by default; add `'use client'` only when needed (event handlers, hooks, browser APIs)
- Tailwind CSS 4 with `@theme inline` for CSS custom properties
- Pipeline scripts (`scripts/`) are ESM `.mjs` with co-located `.test.mjs` files; app code is TypeScript with co-located `.test.ts`/`.test.tsx`
